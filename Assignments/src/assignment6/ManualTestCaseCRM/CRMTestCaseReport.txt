
# **Manual Test Case Report**

## **Customer Relationship Management (CRM) System**

---

## **1. Introduction**

This document contains the **manual test cases** for the Customer Relationship Management (CRM) system.
The purpose of testing is to verify that all CRM features work correctly as per requirements and are user-friendly, secure, and error-free.

Manual testing is performed **without using any automation tools**. All test cases are executed manually by the tester.

--------------------------------------------------------------------------------------------------------------------------------
## **2. Scope of Testing**

The following modules of the CRM system are tested:

* User Login
* User Registration
* Customer Management
* Search Functionality
* Update Customer Details
* Delete Customer Records
* Logout Functionality

--------------------------------------------------------------------------------------------------------------------------------

## **3. Test Environment**

| Item           | Description         |

| Application    | CRM Web Application |
| Testing Type   | Manual Testing      |
| Platform       | Windows             |
| Browser        | Google Chrome       |
| Tester         | Manual Tester       |
| Testing Method | Black Box Testing   |

--------------------------------------------------------------------------------------------------------------------------------
## **4. Test Case Format**

Each test case contains:

* Test Case ID
* Test Case Description
* Test Steps
* Expected Result
* Actual Result
* Status (Pass/Fail)
