# **Manual Test Case Report**

## **Automated Banking Website**

---

## **1. Introduction**

This document contains the **manual test cases** for the Automated Banking Website.
The purpose of testing is to ensure that users can **securely perform banking operations** such as login, fund transfer, balance check, and transaction management without errors.

All testing is performed **manually without using any automation tools**.

---

## **2. Scope of Testing**

The following modules of the Automated Banking Website are tested:

* User Registration
* User Login
* Account Balance Inquiry
* Fund Transfer
* Transaction History
* Bill Payment
* Logout Functionality

---
